# 🤖 ProcureAI Agent 训练指南

> 给组员看 — 为什么要训练、怎么训练、需要你们做什么

---

## 一、先搞懂：Agent 现在怎么工作的

当用户在前端输入"找德国的玻璃胶供应商，IATF 认证"，Agent 做三件事：

```
① 意图解析（parser.py）
   "德国" → country=Germany
   "玻璃胶" → category=glassAdhesive
   "IATF认证" → certifications=["IATF 16949"]

② 搜索供应商（retriever.py）
   本地 24 家供应商库 + DuckDuckGo + 动态爬虫

③ 打分排序（ranker.py）
   Henkel AG: 97 分 ← 为什么 97？因为规则写的
   Sika:      94 分
   SaarGummi: 78 分 ← 品类不匹配，降分
```

**问题在第三步：打分规则是人手写的，不一定准。**

---

## 二、训练做什么：让 Agent 自己学会打分

### 不是改模型参数

我们不能 fine-tune GPT-4o（API 调用的模型不能改权重）。

### 是优化 prompt（提问方式）

```
训练前：
  Agent 问 LLM："给这些供应商打分"
  → 不一定符合采购人员的判断标准

训练后：
  Agent 问 LLM："给这些供应商打分。注意：IATF 认证是硬性要求，
   没有的直接排除；紧急采购时交期优先于价格；德国本地供应商+
   10分..."
  → 跟采购人员的判断越来越接近
```

### 打个比方

```
你是数学教授（GPT-4o），本来就会算
但每个人想要的结果不一样：
  - 采购 A 认为"价格最重要"
  - 采购 B 认为"认证最重要"

DSPy 做的事：
  给你看 50 个案例（"这种情况选 A，那种情况选 B"）
  → 你总结出规律
  → 以后遇到新案例你会按规律判断
```

## 三、训练需要什么数据

### 格式（一条训练数据）

```json
{
  "query": "找德国的挡水条供应商，IATF认证必须，两周内到货",
  "expected_supplier": "Cooper Standard France SAS",
  "expected_score": 90,
  "reason": "挡水条专业、有IATF认证、法国就近、交期稳定"
}
```

### 只要三样东西

| 字段 | 说明 | 例子 |
|------|------|------|
| `query` | 采购需求（自然语言，怎么搜就怎么写） | "找德国玻璃胶供应商 IATF 认证" |
| `expected_supplier` | 你觉得最应该推荐的供应商 | "Henkel AG & Co. KGaA" |
| `expected_score` | 大概的匹配度（0-100） | 95 |
| `reason` | 为什么选它（一句话） | "IATF认证、全球第一、价格合理" |

## 四、数据从哪来

### 方式 A：找采购人员（最推荐）

带着 Agent 去问他们：

```
你最近一次采购——搜的什么？最后选了谁？为什么？
```

每人问 5-10 个场景，记下来就是训练数据。

### 方式 B：凭经验写（次选）

我们自己假设场景。比如：
- "紧急采购玻璃胶，3天内必须到" → 选交期最短的
- "只要 IATF 认证的挡水条" → 排除没认证的
- "预算不限，要最好的" → 选评分最高的

```bash
cd backend
python trainer/dspy_optimizer.py --mode pre --input training_examples.json
```

输出：

## 六、训练覆盖哪些维度

| 维度 | 训练让它学会 |
|------|------------|
| **品类匹配** | 搜"玻璃胶"不应该推荐"密封条"供应商 |
| **认证要求** | "IATF 必须" → 没认证的降分 |
| **地区偏好** | "德国" → 德国供应商加分 |
| **价格敏感** | "越便宜越好" → 价格优先 |
| **交期优先** | "3天必须到" → 交期第一 |
| **综合考量** | "预算€15以内，IATF认证，最好德国" |

## 七、现在需要你们做什么

**每人写 10-15 条训练数据**，格式照着第三节来。

覆盖不同场景：
- 不同品类（玻璃胶、挡水条、密封条、玻璃原片...）
- 不同约束（价格敏感、交期优先、认证必须、综合考量）
- 不同地区（德国、法国、意大利、不限）
- 极端情况（条件非常严格→可能没有匹配）

交给我后我跑 DSPy，Agent 的推荐就会跟真人判断越来越接近。

## 八、常见问题

**Q：我不知道 expected_score 该给多少？**
A：大概就行。95 以上 = 完美匹配，80-94 = 比较匹配，60-79 = 勉强可以，60 以下 = 不太合适。

**Q：24 家供应商我不认识怎么办？**
A：看 `backend/data/suppliers.json`，里面有每家供应商的品类、国家和描述。根据这些判断就行。

**Q：训练完就能看到效果吗？**
A：能。训练后 Agent 会输出 `optimized_prompt.txt`，可以看到 prompt 怎么变的。而且后续搜索的推荐结果会更符合人工判断。
